- **kumatalent.com** = marketing site. Vercel project **kuma-talent-web** (account mrputridsden, Hobby plan). Repo `C:\Users\andre\repos\kuma-talent-web`, single static `index.html`. Deploy via Vercel CLI: `vercel deploy --prod --yes --cwd "C:\Users\andre\repos\kuma-talent-web"` (NOT git push — see [[reference_deploy_mechanisms]]). Aliases to `kuma-talent-web.vercel.app`.
- **app.kumatalent.com** = HireSignal app. DO App Platform app `hiresignal-pqsee.ondigitalocean.app` (region ATL1). Custom domain added in the app's Settings → Domains, set PRIMARY. HireSignal and Kuma Talent are **separate products** — don't link the marketing CTAs to HireSignal copy; CTAs (Apply / Post a Job) point to `https://app.kumatalent.com`.
- **DNS**: authoritative on **DigitalOcean** (`ns1/2/3.digitalocean.com`); registrar is **GoDaddy** (delegation points to DO). Zone records: apex `A → 216.198.79.1` (Vercel), `www CNAME → <hash>.vercel-dns-017.com` (Vercel), `app CNAME → hiresignal-pqsee.ondigitalocean.app`, MX → secureserver.net (GoDaddy email), SPF TXT. Keep MX untouched on any cutover.
- **Canonical URL is the apex** `kumatalent.com` (set as "production/serve" in Vercel); `www` redirects to it.

**HireSignal app details:** source at `C:\Users\andre\repos\hiresignal` (PHP/Apache, Dockerfile). Deploys to DO App Platform via **git push to `origin` = `Lucca-Tech/hiresignal` master** (`.do/app.yaml` has `deploy_on_push: true`; there's also an `andy` remote = personal fork, not the deploy source).

**Operating model (2026-06-24, see `docs/operating-model.md` in the repo):** one isolated clone per client (own DO app + own DB). **We manage the plumbing via env vars** (SMTP creds, ANTHROPIC_API_KEY, ADMIN_PASSWORD, DATABASE_URL, DEV_MODE, PUBLIC_BASE_URL); **the client only sees branding + report recipients** in the admin Settings panel. Settings UI was stripped of SMTP/DEV/API fields. `recruiter_email` is admin-controlled (removed from the env-override list in config.php). **Persistence:** a Postgres dev DB is attached as resource `db`, exposed via env `DATABASE_URL=${db.DATABASE_URL}` (bindable ref, no plaintext password). `db.php` falls back **silently** to ephemeral JSON if the DB is unreachable — a green deploy does NOT prove DB connection; verify by changing a setting + redeploy + check it survives. The **logo is stored as base64 in the settings doc (DB)**, GD-downscaled to 600px, input capped at 1MB/2000px to avoid memory spikes — NOT on the ephemeral container disk. Invite links are built from `PUBLIC_BASE_URL` in `config.php` (`$_st['public_base_url'] ?? 'https://app.kumatalent.com'` — default changed from www to app on 2026-06-24). **No admin-UI field** exists for the base URL, so it uses the code default; to switch domains either edit config.php or add a Settings field. Secrets live as DO App-Level env vars (SMTP via Gmail kumatalent49@gmail.com, ANTHROPIC_API_KEY, ADMIN_PASSWORD). The marketing repo carries a Vercel redirect (`vercel.json`) `/index.php → app.kumatalent.com/index.php` to rescue already-distributed `www` invite links (query string preserved).

**Marketing site = "Warm Den" redesign (2026-06-25):** earth-tone palette (terracotta `#B5694A`, ochre, sage, espresso, cream) + Fraunces display font; real Kuma bear logo (single-line bear + signal wave) in `assets/logo/` (isotipo-light = black for light bg, isotipo-dark = white for dark bg, logo-principal[-dark]). Bear logo draws-in via CSS clip-path; a signal-wave SVG draws itself in the hero. ALL CTAs are a single **"Join Kuma"** that scrolls to an on-site `#join` form — the marketing site no longer links to HireSignal at all, and **no email/mailto appears anywhere** (per the owner).

**Lead/CV capture (free, no respondent login, no email exposed):** the `#join` form has **Candidate / Employer tabs** (Candidate shows a CV file-upload; Employer shows Company). On submit, JS reads the CV as base64 and `fetch(..., {mode:'no-cors'})` POSTs JSON to a **Google Apps Script web app** (`APPS_SCRIPT_URL` in index.html, `…/exec`). The Apps Script (bound to the Sheet **"Kuma Talent Database"** `18Q4_asLlyBO0c752nVfErHxgRKuRXDn8-LbWkJruin0`, deployed Execute-as-Me / access **Anyone**) saves the CV to Drive folder *"Kuma Talent - CVs"* and appends a row (Date, Name, Email, I am a, Company, Message, CV link). Chosen over Google Forms because Forms' file-upload **forces respondent Google sign-in**; over Web3Forms because its free plan **can't attach files**. Gotcha: opening Apps Script with multiple Google accounts logged in throws "unable to open the file" (`authuser=1` mismatch) — fix is an **incognito window signed in only as kumatalent49**. CORS: Apps Script returns no CORS headers, so the form uses `no-cors` (opaque response) and shows optimistic success.

**Gotcha (cost real time this session):** removing + re-adding a Vercel custom domain **resets its Let's Encrypt cert issuance and can rate-limit it** — `www` got stuck "Invalid Configuration / Generating SSL" and TLS handshake kept failing ("unsupported protocol" / SSPI). Fix that actually worked: stop churning the domain, and **set the apex as the serving/production domain** (apex cert issues fast) instead of waiting on `www`. Don't move nameservers to Vercel — that would orphan the app CNAME + MX. Adding `ns*.vercel-dns.com` records *inside* the DO zone is inert (delegation is at the registrar).

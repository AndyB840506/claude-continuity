AI-generated giveaways must be avoided in EVERY client-facing deliverable, not just
web pages: logos, standalone graphics, and written work (audits, proposals, copy)
all need an explicit anti-AI-tell pass before being called done.

**Why:** [[feedback_premium_web_design]] already bans the generic AI skeleton, but
only for web pages (`web-page-kit/docs/design-guide.md` Rule 0). The user's 2026-06-29
feedback during the `freelance-gig` skill build generalized this: "los outputs no
pueden dar indicios de uso de IA... en especial en las creaciones visuales, como
paginas web, diseño de logos, etc." — for freelance deliverables specifically, an
AI-detectable output risks the client relationship, not just aesthetic quality.
Demands "outside the box" thinking per piece, not template reuse.

**How to apply:** When producing any client-facing visual or written deliverable
(freelance gigs, but the principle generalizes): (1) for web/HTML work, read and
apply `design-guide.md` Rule 0 + its anti-generic checklist live — don't re-derive
a thinner version elsewhere; (2) for logos/graphics, ban the same telltale
font/palette defaults and avoid generic AI-stock-image look; (3) for written work,
check against formulaic AI-writing tells (em-dash hedging, "not just X, it's Y,"
triplet lists, generic transitions, qualifier-stacking); (4) name one concrete
client-specific angle before generating, and end with an explicit self-check
("does this look/read like a template or like AI on autopilot?") before declaring
done. Encoded in `freelance-gig/workflows/build.md` and
`docs/estandar-de-entregables.md` section 8.

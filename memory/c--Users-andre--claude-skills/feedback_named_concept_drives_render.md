When a redesign has a **named concept** (e.g. Andyfreelancer = "boutique-showroom that sells experiences, not products"), the render must *embody* that concept — not default to the generic AI skeleton or, worse, the wrong metaphor. On Andyfreelancer a grid layout read as "engineering" and was rejected **twice** ("quedo muy enmarcada la parte de ingenieria"; "sigue siendo ingenieria... no vendemos productos sino experiencias"). Pure-engineering framing was explicitly reserved for a *different* brand (Lucca Tech).

**Why:** the concept is the differentiator (Rule 0). A grid/stat-row/engineering look contradicts a boutique/experience brand and makes the page feel like every other AI page. The named concept decides palette, type, layout, and the signature metaphor.

**How to apply:**
- Re-run Rule 0 from the *named concept*: let it pick the mood, not a default skeleton. Boutique/experience → editorial, atmospheric, signature interactive field (e.g. the world map). Avoid grids, big-number stat rows, em-dash-heavy copy (AI tells).
- If the user is unfamiliar with a proposed typeface, build a **live HTML specimen** with the real Fontshare faces — don't render a Canva mockup (Canva bakes in un-editable garbled microtext/typos).
- When a metaphor gets rejected as "the wrong vibe," change the *architecture* (layout, type scale, signature element), not just colors/fonts. See [[feedback_reskin_vs_overhaul]] and [[feedback_premium_web_design]].

Once you've read enough of a reference file to understand its architecture or pattern, stop reading and start building.

**Why:** During the BTQ website session, I read the yoguinna HTML file multiple times (full 727 lines) after already having the pattern I needed. The user called it out directly: "Estas construyendo para mi o para yoguinna?" Time spent studying = time not delivering.

**How to apply:** The threshold is: "Can I write the target file now?" If yes, write it. A reference file gives you a pattern — you don't need to memorize it line by line. Read enough to understand structure + tokens + key patterns, then build. If something specific is missing mid-build, spot-read that section only.

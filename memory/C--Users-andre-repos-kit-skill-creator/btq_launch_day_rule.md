Episodes launch on **Spotify Sunday at 8:00 PM Colombia time**. The audience listens Sunday night in "prep for the week" mode — "Aplicable Hoy" lands right as the work week begins, hours away.

**Why:** User confirmed this directly during the EP.016 pipeline run (2026-06-07), after I found THREE conflicting calendar sources in the kit (this memory said "Monday midnight, 3-day Dom/Lun/Mié"; `btq-project/SKILL.md` §13 and `episode-launch/SKILL.md` separately said "3-day Dom/Lun/Mié"; only §10 of `btq-project/SKILL.md` had the real rule). User's exact words: "we will be launching each episode on sunday 8PM since a lot of folks are getting ready to begin their weeks and that time is the best since they're free."

**How to apply:** When generating social plans for any BTQ episode, use this 4-day structure (canonical source: §10 of `btq-project/SKILL.md`):
- **Jueves**: Intriga — sembrar la pregunta, no revelar el tema completo
- **Sábado**: Calentamiento — behind the scenes mientras graba (contenido orgánico, costo cero)
- **Domingo 8:00 PM**: Lanzamiento — episodio disponible, CTA directo
- **Martes**: Refuerzo/herramienta — profundiza para quien ya escuchó, engancha a quien no

This superseded both the "Monday midnight" version originally in this memory AND the "Wednesday 3-day" version that had crept into `episode-launch/SKILL.md` and `btq-project/SKILL.md` §13 — all three were corrected in the same session (2026-06-07, EP.016 pipeline close).
